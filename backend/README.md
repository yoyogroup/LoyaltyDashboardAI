# Yoyo Loyalty Portal — Auth Backend

Serverless OTP + JWT auth. Built with **AWS SAM** (CloudFormation under the
hood — matches the Yoyo Engineering Guardrails AWS-first policy).

```
Browser ──POST /auth/request-otp──► API Gateway ──► Lambda ──► DynamoDB (OTP, TTL 10min)
                                                        └────► Resend  ──► inbox
Browser ──POST /auth/verify-otp ──► API Gateway ──► Lambda ──► DynamoDB (verify + delete)
                                                        └────► returns JWT (30-day)
```

## What gets deployed

| Resource | Why |
|---|---|
| `AuthApi` (API Gateway HTTP API) | Two POST routes; CORS locked to the portal origin; throttled 10 rps / 20 burst |
| `RequestOtpFn` (Lambda, Node 20, arm64) | Validates email, generates OTP, calls Resend |
| `VerifyOtpFn` (Lambda, Node 20, arm64) | Verifies OTP, issues 30-day JWT |
| `OtpTable` (DynamoDB) | One row per pending OTP, **TTL = 10 min**, encrypted at rest, PITR on |
| `RateLimitTable` (DynamoDB) | Rate limit marker per email, **TTL = 60 s** |
| `ResendApiKeySecret` (Secrets Manager) | Holds your Resend API key — populate post-deploy |
| `JwtSigningSecret` (Secrets Manager) | 64-byte HS256 key — auto-generated by CFN |
| `AuthApiLogGroup` (CloudWatch Logs, 90-day retention) | Access logs |

Costs at low volume: well within AWS free tier. Realistic ceiling for a
20-user merchant pilot is < $1/month.

## Prereqs (one-off)

1. **AWS CLI** ≥ 2.15 configured with credentials for the Yoyo AWS account:
   ```bash
   aws --version
   aws sts get-caller-identity
   ```
2. **AWS SAM CLI** ≥ 1.115:
   ```bash
   brew tap aws/tap && brew install aws-sam-cli
   sam --version
   ```
3. **Resend account** with **`yoyogroup.com`** (and/or `yoyorewards.com`) verified as a
   sending domain. You'll need:
   - The DNS records published (Resend will give you SPF/DKIM TXT records — add them in your domain registrar)
   - An API key with **`emails:send`** permission

## Deploy

```bash
cd loyalty-portal/backend

# Install Lambda dependencies (SAM uses these via CodeUri:src/)
cd src && npm ci --omit=dev && cd ..

# First-time deploy — interactive (region, stack name, confirm changes)
sam deploy --guided
#   Stack Name:              yoyo-loyalty-auth
#   Region:                  eu-west-1   (closest to ZA for users in Europe; use af-south-1 if you need lower latency to JHB)
#   Confirm changes:         y
#   Allow IAM role creation: y
#   Disable rollback:        n
#   Save args to samconfig:  y

# Subsequent deploys
sam deploy
```

Take note of the **Outputs** at the end:

```
ApiBaseUrl       https://abc123xyz.execute-api.eu-west-1.amazonaws.com
ResendSecretArn  arn:aws:secretsmanager:eu-west-1:…:secret:yoyo-loyalty/resend-api-key-…
JwtSecretArn     arn:aws:secretsmanager:eu-west-1:…:secret:yoyo-loyalty/jwt-signing-key-…
```

## Post-deploy: populate the Resend API key

The CFN stack creates an **empty** Secrets Manager entry — populate it
manually so the key never lives in git or in a CFN parameter (which would
appear in CloudFormation history).

```bash
# Paste your real Resend API key when prompted
read -rsp "Resend API key: " RESEND_KEY && echo
aws secretsmanager put-secret-value \
  --secret-id yoyo-loyalty/resend-api-key-yoyo-loyalty-auth \
  --secret-string "$RESEND_KEY"
unset RESEND_KEY
```

(Replace the secret-id suffix with whatever `--stack-name` you used.)

## Wire the front-end

1. Open `loyalty-portal/assets/config.js`.
2. Set `apiBase` to the `ApiBaseUrl` output.
3. Set `AUTH_ENABLED: true`.
4. Commit + push → GitHub Actions deploys to Pages.

```js
window.YOYO_CONFIG = Object.freeze({
  AUTH_ENABLED: true,
  apiBase: 'https://abc123xyz.execute-api.eu-west-1.amazonaws.com',
  allowedDomains: Object.freeze(['yoyogroup.com', 'yoyorewards.com']),
  sessionDays: 30,
});
```

## Smoke test

```bash
# 1) Request an OTP — should return {ok:true} and you get an email
curl -X POST "$API/auth/request-otp" \
  -H "Content-Type: application/json" \
  -d '{"email":"you@yoyogroup.com"}'

# 2) Verify with the 6-digit code from the email — returns a JWT
curl -X POST "$API/auth/verify-otp" \
  -H "Content-Type: application/json" \
  -d '{"email":"you@yoyogroup.com","code":"123456"}'

# 3) Disallowed domain — returns {ok:true} but no email sent (don't leak)
curl -X POST "$API/auth/request-otp" \
  -H "Content-Type: application/json" \
  -d '{"email":"random@gmail.com"}'
```

## Operations

### Audit log
Every event lands in CloudWatch Logs as one-line JSON:

```bash
aws logs tail /aws/lambda/yoyo-loyalty-request-otp-yoyo-loyalty-auth --follow
aws logs tail /aws/lambda/yoyo-loyalty-verify-otp-yoyo-loyalty-auth --follow
```

Useful filters:
- `evt=otp.request.sent`     — successful OTP send
- `evt=otp.request.rate-limited` — abuse signal
- `evt=otp.verify.success`   — successful sign-in
- `evt=otp.verify.lockout`   — 5 failed codes for one email

Email addresses appear **only as SHA-256 hashes** (`email_hash`) — POPIA safe.

### Rotate the JWT signing key
```bash
aws secretsmanager rotate-secret --secret-id yoyo-loyalty/jwt-signing-key-yoyo-loyalty-auth
```
Existing JWTs are invalidated immediately — every user has to sign in again.
Plan this for a quiet window. (Recommended quarterly per the guardrails.)

### Force-logout a single user
Delete their OTP record (clears any pending code) — and since the JWT
signing key isn't per-user, the only ways to invalidate one user's session
today are:
  (a) rotate the JWT key (logs out everyone), or
  (b) add a server-side session blocklist (small extension — ask Claude to wire it).

### Tear down
```bash
sam delete --stack-name yoyo-loyalty-auth
```
This deletes Lambdas, API Gateway, and DynamoDB tables. Secrets are kept
in a "scheduled deletion" state for 30 days (AWS safeguard); to nuke
immediately:
```bash
aws secretsmanager delete-secret --secret-id yoyo-loyalty/resend-api-key-yoyo-loyalty-auth   --force-delete-without-recovery
aws secretsmanager delete-secret --secret-id yoyo-loyalty/jwt-signing-key-yoyo-loyalty-auth --force-delete-without-recovery
```

## Future hardening (when the portal moves to a custom domain)

Today the front-end is on `yoyogroup.github.io` and the API is on
`*.execute-api.amazonaws.com` — different origins. That means we use a
JWT-in-localStorage pattern, which is fine given:
  - CSP locks the script origins
  - All DOM rendering uses `textContent` / `createElement` (no XSS sinks)
  - 30-day rolling expiry

When you move to `loyalty.yoyogroup.com` (front-end) + `api.loyalty.yoyogroup.com`
(API Gateway custom domain), switch to **HttpOnly + Secure + SameSite=Lax** cookies
for the JWT. That eliminates the XSS-steal vector entirely. Code change is ~30
lines (move from `localStorage.setItem` to `Set-Cookie` server-side + remove
the `Authorization: Bearer` header on the client).
